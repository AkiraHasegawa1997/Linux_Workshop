‣ 変数
変数は値や文字列に名前をつけて保持しておく仕組みのことです．
 名前=値 という形式のコマンドです．
このとき，名前と=，値の間にスペースを入れないように注意してください．
使用例1 : 変数VARに"Hello, World"を代入．

	 $ VAR="Hello, World" 

変数にアクセスするためには名前の前に $ をつけます．
使用例2 : 上で定義したVARを echo で出力．

	 $ echo $VAR 

変数の一部を他の文字に置き換えるには${VAR/置換前/置換後}とします．
使用例3 : 変数 VAR のWorldをLinuxに置き換えて出力する．

	 $ echo ${VAR/World/Linux}

‣ 環境変数
シェル起動時に定義されている特殊な変数があります．
使用例1 : 環境変数を一覧表示

	 $ env 

 新しい端末を開いてから 次の例を実行してみましょう．
使用例2 : 環境変数PATHを表示し空白で初期化したあとに
          コマンドを実行

	 $ echo $PATH 
	 $ PATH="" 
	 $ echo PATH 
	 $ ls 

おそらく ls が実行できなくなっていると思います．
変数 PATH はコマンドで実行されるプログラムの
場所を保持している変数です．
変数 PATH を初期化するとこの例のように一部のコマンドが
使えなくなってしまいます．

‣ xargsと変数
今回使用しているシェルそのもののコマンド名は bash といいます．
 bash -c を使うとコマンドを実行することができます．
また， ; を使うと複数のコマンドを1行で順に実行できます．
使用例 :  bash -c を使って ls ， pwd ， ls を順に実行

	 $ bash -c "ls; pwd; ls" 

このことを使うと，xargsからbashの変数へ値を渡してコマンドを実行することができます．
使用例 : flacファイルをmp3ファイルに変換

	 $ ls *.flac | xargs -t -n1 -I{} bash -c 'VAR={}; ffmpeg -i VAR ${VAR/.flac/.mp3}' 

補足 : 上の例は込み入っているので詳しく説明します．
まず，パイプの左側で .flac で終わるファイルを全て出力しています．
パイプの右側では，その出力のうち1つだけを {} に代入し
 bash のコマンドへ渡しています．
そして，xargsの {} がbashの変数 VAR に代入され，処理が行われます．
